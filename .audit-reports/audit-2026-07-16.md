## 详细 Issue

### P1-1 缺少 CODEOWNERS（治理）
- **文件**：`.github/CODEOWNERS`（缺失）
- **问题**：外部贡献者 PR 无自动 reviewer 指派，与 v5 审计"治理三件套"目标不符（SECURITY + CONTRIBUTING 已有，CODEOWNERS 缺）
- **失败场景**：fork 提 PR 无人审，或误合未授权改动
- **修复**：加 `.github/CODEOWNERS` 指派 `@gandli`
- **回归测试**：无（配置项）；建议 PR 模板引用
- **工作量**：5 min

### P1-2 mypy 未纳入 CI gate（CI 硬化）
- **文件**：`.github/workflows/ci.yml:47`
- **问题**：`pyproject.toml` 已声明 `mypy>=1.11` 为 dev 依赖，但 CI 只跑 ruff + pytest，**类型检查不卡合并**
- **失败场景**：类型回归（如 `renderer.py` 返回 `list[str]` 签名漂移）静默合入
- **修复**：CI 加 `uv run mypy scripts/vmware_lib` step（严格，零错误）
- **回归测试**：CI 自身
- **工作量**：10 min

### P1-3 bandit 未纳入 CI gate（SAST）
- **文件**：`.github/workflows/ci.yml:47`
- **问题**：`pyproject.toml` 声明 `bandit>=1.7`，但 CI 不跑 SAST。urlopen 虽有 `# nosec B310`，但无自动检查会退化
- **失败场景**：未来新加危险调用（subprocess shell / 未校验 urlopen）漏检
- **修复**：CI 加 `uv run bandit -r scripts/vmware_lib -c pyproject` step
- **回归测试**：CI 自身
- **工作量**：10 min
